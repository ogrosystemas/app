======================================================================
MEMORIAL DESCRITIVO: ERP MASTER MERCADO LIVRE (DOMINGO 21H)
======================================================================

1. INFRAESTRUTURA (Onde o sistema mora)
-------------------------------------------
- Provedor: VPS Contabo (Configurada).
- Painel: aaPanel (Instalado).
- Banco de Dados: PostgreSQL (via Docker ou nativo no aaPanel).
- Runtime: Node.js (Next.js 15).
- Gerenciador de Processos: PM2.

2. ARQUITETURA DO SOFTWARE
-------------------------------------------
- Framework: Next.js 15 (App Router).
- Estilização: Tailwind CSS.
- ORM/DB: Prisma (PostgreSQL).
- Autenticação: Auth.js (ou similar) com níveis de permissão.
- Integração: API Oficial do Mercado Livre (OAuth 2.0).

3. MÓDULOS CORE (A "Vibe" do Sistema)
-------------------------------------------
A. SISTEMA DE LICENCIAMENTO (O Cérebro Financeiro):
   - Controle de mensalidades, bloqueio automático por inadimplência.
   - Painel Admin para gerar chaves de acesso para novos clientes.

B. MÓDULO SAC (Atendimento):
   - Integração com mensagens do Meli.
   - Histórico de conversas e agilidade nas respostas.

C. MÓDULO WMS (Estoque/Depósito):
   - Gestão de entrada/saída, separação de pedidos.
   - Sincronização em tempo real com o estoque do Mercado Livre.

D. MÓDULO FINANCEIRO:
   - Dashboard de lucro real (descontando taxas Meli e impostos).
   - Fluxo de caixa e relatórios de vendas.

-------------------------------------------

4. ATALHOS DE SOBREVIVÊNCIA
---------------------------
- Ctrl + E: Comando universal do Emmet para expandir código (caso o Tab falhe).
- html:5 + Tab: Gera a estrutura inicial do HTML5.
- Ctrl + P: Busca rápida de arquivos dentro do projeto.
- Ctrl + Shift + P: Abre o Package Control (Comandos do Sublime).

5. FLUXO DE DOMINGO (21H)
-------------------------
1. Receber código do Claude (Módulos: SAC, WMS, FINANCEIRO, LICENCIAMENTO).
2. Colar no Sublime (Usar abas para organizar cada módulo).
3. Salvar os arquivos na pasta local do projeto.
4. Subir para o aaPanel via Gerenciador de Arquivos ou SFTP.
5. Rodar "npm install" e "npm run dev" no Terminal da VPS.
